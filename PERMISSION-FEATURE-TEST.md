# 文档权限功能测试指南

## 测试目标

验证机器人在创建文档时能够自动将对话参与者添加为协作者。

## 前置条件

### 1. 权限配置

访问飞书开放平台，确保机器人有以下权限：

```
✅ docx:document:create       - 创建文档
✅ docx:document:write        - 修改文档内容
✅ docx:document:permission   - 管理文档权限 ✨ 必需
✅ contact:user:readonly      - 读取用户信息
✅ contact:group:readonly     - 读取群组信息 ✨ 必需
✅ contact:group.member:readonly - 读取群成员 ✨ 必需
✅ im:message:send_as_bot    - 发送消息
```

### 2. 测试环境

- 机器人已加入测试群（群聊场景）
- 或已与机器人建立私聊（私聊场景）
- OpenClaw已运行并加载最新插件

## 测试场景

## 常见问题排查

### 问题1：没有看到权限提示

**可能原因**：
1. context没有正确传递
2. chatId为空

**排查方法**：
```bash
# 检查消息对象结构
console.log('Message:', JSON.stringify(message, null, 2));
# 应该有 chat_type 和 chat_id 字段
```

**解决方法**：
确保消息对象包含必要字段：
```javascript
{
  chat_type: 'group',  // 或 'private'
  chat_id: 'oc_xxx',   // 群ID
  sender_id: 'ou_xxx'  // 用户ID
}
```

---

### 问题2：成员数量不对

**可能原因**：
1. 飞书API返回不完整
2. 权限配置不完整
3. 群成员被过滤（机器人、服务号等）

**排查方法**：
```bash
# 查看日志中的成员数量
📋 从群聊获取到 5 个成员
```

**解决方法**：
1. 检查飞书开放平台权限配置
2. 确认群成员类型（只添加真实用户）
3. 检查API返回的数据

---

### 问题3：部分成员无法访问文档

**可能原因**：
1. 权限设置失败
2. 用户没有飞书账号
3. 用户被禁用

**排查方法**：
1. 查看飞书日志
2. 在文档设置中手动添加用户测试
3. 检查用户账号状态

**解决方法**：
- 如果是权限设置失败，检查机器人权限配置
- 如果是用户账号问题，让用户联系管理员

---

### 问题4：日志显示"添加文档协作者失败"

**可能原因**：
1. 缺少必要的API权限
2. 权限API调用失败
3. 网络问题

**排查方法**：
```bash
# 查看完整错误信息
console.error('添加文档协作者失败:', error);
```

**解决方法**：
1. 检查飞书开放平台权限配置
2. 确认有 `docx:document:permission` 权限
3. 检查网络连接

    console.log('✅ 所有测试通过！');
}

if (require.main === module) {
