# 飞书知识库智能助手 - 使用说明

## 快速开始

### 在飞书群中使用

1. 将机器人添加到飞书群聊
2. @机器人 发送消息或命令
3. 机器人会自动识别您的意图并响应

### 基础命令

```
/help      # 查看帮助信息
/find      # 查找文档
/search    # 搜索内容
/create    # 创建文档
/todo      # 添加待办
/stats     # 查看统计
/index     # 重建索引
```

## 功能详解

### 1. 智能文档定位

**用途**：快速找到目标文档的精确链接

**使用方式**：
- 自然语言：`产品需求文档在哪里`
- 关键词查找：`查找技术架构文档`
- 命令方式：`/find 产品需求`

**返回信息**：
- 文档标题
- 所属知识库
- 文档链接
- 最后更新时间

**示例**：
```
您: 产品需求文档在哪里
机器人: 📄 找到文档：产品需求文档 - 飞书知识库智能助手
📍 位置：产品文档
🔗 链接：https://xxx.doc
📅 更新：2024/01/02
```

### 2. 高效内容检索

**用途**：搜索知识库内容，返回信息摘要

**使用方式**：
- 自然搜索：`搜索API接口相关内容`
- 主题查询：`关于项目进度的文档`
- 命令方式：`/search 技术架构`

**返回信息**：
- 相关文档列表
- 内容摘要
- 相关度评分
- 相关文档推荐

**示例**：
```
您: 搜索技术架构相关内容
机器人: 🔍 找到 2 个与"技术架构"相关的结果

1. 技术架构设计文档
   📝 本文档描述了系统的整体架构设计和技术选型...
   🔗 [查看文档](https://xxx.doc)
   ⭐ 相关度: 85%
```

### 3. 自动智能归档

**用途**：新建文档并自动分类归档

**使用方式**：
- 创建新文档：`新建产品需求：新增用户权限管理功能`
- 命令方式：`/create 会议记录 - 项目周会`

**自动分类**：
- 技术文档 → 技术知识库
- 产品文档 → 产品知识库
- 会议记录 → 会议知识库
- 项目文档 → 项目管理知识库

**分类依据**：
- 标题关键词分析
- 内容语义识别
- 置信度评分

### 4. 会议协作自动化

**用途**：自动提取待办、生成会议文档

**添加待办**：
```
您: /todo 需要完成用户权限接口开发
机器人: ✅ 待办事项已添加
📄 文档：当前项目文档
✨ 待办内容：需要完成用户权限接口开发
```

**生成会议文档**：
```
您: /create 本周项目例会:讨论了项目进度和风险
机器人: ✅ 会议文档已生成
📄 文档标题：本周项目例会
📅 会议时间：2024/01/05
👥 参与人数：5人
✅ 待办事项：3个
```

## 高级技巧

### 自然语言识别

机器人能智能识别您的意图：

```
"帮我找一下API文档"           → 文档定位
"搜索关于性能优化的内容"      → 内容搜索
"新建一个bug修复记录"         → 创建文档
"记录一下今天要完成的任务"    → 添加待办
```

### 精确搜索

使用更具体的关键词获得更精准的结果：

```
❌ 模糊搜索: 文档
✅ 精确搜索: 用户权限管理API文档
```

### 批量操作

对于多个相似操作，可以一次性完成：

```
"新建以下文档：1.需求文档 2.设计文档 3.测试文档"
```

## 常见问题

### Q: 机器人找不到我需要的文档怎么办？

A: 尝试以下方法：
1. 使用不同的关键词重新搜索
2. 使用更通用的搜索词
3. 确认文档是否在机器人有权限访问的知识库中

### Q: 如何提高搜索准确度？

A:
1. 使用文档标题中的关键词
2. 避免使用过于宽泛的词汇
3. 结合文档所属分类进行搜索

### Q: 创建的文档被自动分类错了怎么办？

A:
1. 可以手动将文档移动到正确的位置
2. 联系管理员调整分类规则
3. 使用更明确的标题和内容

### Q: 机器人响应很慢怎么办？

A:
1. 检查是否是网络问题
2. 查看服务状态：发送 `/stats`
3. 必要时重建索引：发送 `/index`

## 最佳实践

### 1. 文档命名规范

```
✅ 好的命名：
- 产品需求文档 - 用户权限管理功能
- 技术架构设计 - 支付系统v2.0
- 会议记录 - 2024年第1周项目例会

❌ 不好的命名：
- 文档1
- 会议纪要
- 新建文档
```

### 2. 待办事项格式

```
✅ 清晰的待办：
- 完成用户权限接口开发（截止日期：本周五）
- 编写API文档并提交审核
- 与产品团队确认需求细节

❌ 模糊的待办：
- 做接口
- 写文档
- 对需求
```

### 3. 搜索技巧

1. **使用特定术语**：使用行业术语或产品名称
2. **组合关键词**：`产品 需求 权限`
3. **限定范围**：`在技术文档中搜索 API`

## 团队协作

### 权限管理

- 确保机器人有权限访问相关知识库
- 定期检查和更新权限设置
- 遵循最小权限原则

### 使用规范

1. @机器人时，清楚表达您的需求
2. 重要信息建议直接查看原文档
3. 定期检查机器人的索引更新状态
4. 遇到问题及时反馈给管理员
