## 🔐 安全配置

### 必需的环境变量

```bash
# 🔒 数据加密密钥 (必须设置)
ENCRYPTION_KEY=请生成32位随机字符串
# 生成方法: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# 🔑 飞书机器人凭证
FEISHU_APP_ID=your_app_id
FEISHU_APP_SECRET=your_app_secret

# 🔐 事件回调验证 (可选)
FEISHU_VERIFICATION_TOKEN=your_token
FEISHU_ENCRYPT_KEY=your_key
```

### 安全文件权限

```bash
# 设置安全的文件权限
chmod 600 .env                    # 只有所有者可读写
chmod 600 data/                   # 数据目录
chmod 700 scripts/*.sh             # 脚本只有所有者可执行
```

### Git安全保护

```bash
# 安装Git安全钩子
bash git-hooks/install.sh

# 钩子会自动检查：
# - 防止提交敏感文件
# - 检测代码中的凭证模式
# - 警告大文件提交
```

## 💡 安全最佳实践

### ✅ 推荐做法

1. **凭证管理**
   - 使用强密码和随机字符串
   - 定期更换密钥和凭证
   - 不同环境使用不同凭证
   - 永远不要在代码中硬编码

2. **权限控制**
   - 遵循最小权限原则
   - 只申请必需的飞书权限
   - 定期审查权限设置
   - 撤销不再使用的权限

3. **数据保护**
   - 敏感数据加密存储
   - 使用安全的通信协议
   - 定期备份重要数据
   - 及时清理临时文件

4. **监控审计**
   - 定期检查API调用日志
   - 监控异常访问行为
   - 记录重要操作日志
   - 建立安全事件响应流程

### ❌ 避免做法

- ❌ 在代码中硬编码凭证
- ❌ 将凭证提交到版本控制
- ❌ 使用弱密码或默认密钥
- ❌ 忽视安全更新和补丁
- ❌ 在公共网络传输敏感信息
- ❌ 与他人共享凭证文件

## 🆘 安全事件响应

### 发现安全问题时的处理流程

1. **立即响应** (15分钟内)
   - 停止受影响的服务
   - 撤销暴露的凭证
   - 通知相关人员

2. **调查评估** (1小时内)
   - 确定问题范围和影响
   - 分析根本原因
   - 评估损失程度

3. **修复恢复** (4小时内)
   - 实施修复措施
   - 恢复正常服务
   - 验证修复效果

4. **总结改进** (1天内)
   - 编写事件报告
   - 改进安全流程
   - 更新安全策略
