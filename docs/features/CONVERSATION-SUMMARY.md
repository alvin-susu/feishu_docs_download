# 历史对话总结功能

## 功能说明

飞书知识库智能助手现在支持**基于历史对话自动生成文档**，可以根据群聊或私聊的历史消息内容，智能分析并生成结构化的文档。

## 🎯 适用场景

### 场景1：会议记录
在群聊中开完会后，自动总结会议内容：
```
@机器人 /meeting
```
机器人会：
- 获取最近100条消息
- 提取参与者、待办事项、决策
- 生成会议记录文档
- 自动添加所有群成员为协作者

### 场景2：决策记录
重要讨论后，生成决策记录：
```
@机器人 /decision
```
机器人会：
- 提取关键决策内容
- 记录决策者和时间
- 生成结构化的决策文档

### 场景3：对话总结
定期总结聊天内容：
```
@机器人 /summary 产品需求讨论
```
机器人会：
- 分析聊天话题
- 提取关键信息
- 生成总结文档

### 场景4：自定义消息数量
指定使用多少条历史消息：
```
@机器人 /summary 200
```
会使用最近200条消息生成文档。

## 🔐 需要的权限

### 飞书开放平台配置

访问[飞书开放平台](https://open.feishu.cn/)，在权限管理中添加：

```
✅ im:message:readonly      - 读取消息内容（必需）
✅ im:chat                  - 获取聊天信息（必需）
✅ im:conversation          - 获取会话信息（推荐）
```

### 权限说明

| 权限 | 说明 | 用途 |
|------|------|------|
| `im:message:readonly` | 读取消息 | 获取历史对话内容 |
| `im:chat` | 获取聊天信息 | 获取群聊/私聊信息 |
| `im:conversation` | 获取会话信息 | 获取会话详情 |

## 🔧 技术实现

### API调用

#### 1. 获取历史消息
```javascript
GET /im/v1/messages
参数:
  - container_id: chat_id
  - container_id_type: "chat"
  - page_size: 50
  - cursor: 分页游标

响应:
{
  "data": {
    "items": [...],      // 消息列表
    "has_more": true,    // 是否有更多消息
    "page_token": "..."  // 下一页游标
  }
}
```

#### 2. 获取单条消息
```javascript
GET /im/v1/messages/{message_id}

响应:
{
  "data": {
    "message_id": "om_xxx",
    "msg_type": "text",
    "content": "...",
    "sender": {...},
    "create_time": 1234567890
  }
}
```

### 消息解析

支持的消息类型：

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| `text` | 纯文本 | 直接提取文本内容 |
| `post` | 富文本 | 提取所有文本段落 |
| `interactive` | 卡片消息 | 提取卡片中的文本信息 |
| `image` | 图片 | 记录为[图片消息] |
| `file` | 文件 | 记录为[文件消息] |

### 智能分析算法

#### 1. 关键词提取
```javascript
// 使用简单的分词和频率统计
const words = text.split(/\s+/);
const frequency = {};
words.forEach(word => {
    frequency[word] = (frequency[word] || 0) + 1;
});
```

#### 2. 待办事项识别
```javascript
// 匹配待办关键词
const patterns = [
    /(?:待办|todo|任务)[:：]\s*(.+)/i,
    /(?:需要|要)[:：]\s*(.+)/i,
    /(?:请|麻烦)[:：]\s*(.+)/i
];
```

#### 3. 决策识别
```javascript
// 匹配决策关键词
if (/决定|确定|同意|通过|采纳/i.test(text)) {
    // 标记为决策
}
```

## ⚙️ 配置选项

### 消息数量限制

- **默认**: 100条消息
- **最小**: 10条消息
- **最大**: 1000条消息（建议不超过500条）

建议：
- 小群（<20人）：100-200条
- 大群（20-50人）：200-500条
- 超大群（>50人）：500-1000条

### 文档类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| `summary` | 通用总结 | 日常对话总结 |
| `meeting` | 会议记录 | 会议讨论、评审 |
| `decision` | 决策记录 | 重要决策、投票 |
| `discussion` | 讨论总结 | 主题讨论 |

## ⚠️ 注意事项

### 权限要求

1. **必须配置权限**
   - 缺少`im:message:readonly`权限会导致无法获取历史消息
   - 需要在飞书开放平台申请并发布

2. **权限范围**
   - 只能获取机器人所在的对话的消息
   - 无法获取机器人加入之前的消息（受飞书限制）
   - 无法获取已撤回的消息

### 使用建议

1. **消息数量**
   - 不要使用过多消息（建议<500条）
   - 消息过多会导致文档过长，生成时间增加

2. **文档标题**
   - 建议使用有意义的标题
   - 标题有助于后续查找和管理

3. **文档类型**
   - 根据实际场景选择合适的文档类型
   - 会议用`/meeting`，决策用`/decision`

### 性能考虑

- 获取100条消息约需1-2秒
- 获取500条消息约需5-10秒
- 建议在非高峰期使用

## 🔍 故障排查

### 问题1：提示"未找到历史消息"

**可能原因**：
1. 对话中没有消息
2. 机器人刚加入，没有历史消息
3. 权限配置不正确

**解决方法**：
1. 确保对话中有历史消息
2. 检查飞书开放平台权限配置
3. 确保机器人已加入对话

### 问题2：获取的消息数量不对

**可能原因**：
1. 机器人加入时间较晚
2. 部分消息已过期（飞书保留有限期）
3. API限流

**解决方法**：
- 这是正常现象，飞书有消息保留期限
- 重要消息建议及时生成文档保存

### 问题3：生成的文档内容不完整

**可能原因**：
1. 消息内容过于简短
2. 关键词匹配不准确
3. 消息类型不支持

**解决方法**：
- 使用更多消息数量
- 手动编辑补充文档内容
